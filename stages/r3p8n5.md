# Recovery instruction file (schema validation failed — single occurrence)

The validator routed you here because a state file failed schema validation in a way that wasn't a transient miss the retry budget could absorb.

## Your inputs (from stderr)

- `ERROR_CLASS=schema_validation_failed`
- `FAILED_PATH` — the state file that failed validation
- `REASON` — the validation error
- `RUN_DIR`

## Procedure

This recovery class is reserved for v3 — single-occurrence schema failures that the retry mechanism doesn't cover. In v2, validation failures are routed through retry first, then to `retry_exhausted` if the budget runs out.

If you reached this file directly, the validator has a bug. Halt and surface to the user with the `FAILED_PATH` and `REASON`.

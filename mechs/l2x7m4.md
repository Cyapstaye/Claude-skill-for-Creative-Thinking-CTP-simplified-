# Clarify decision

> **Dispatch-forced.** This file is read only when the dispatcher routes you here. Its content cannot be performed without the dispatcher's directives. The bootstrap script invokes you with the user's query in `$ORIGINAL_QUERY`, an optional `--clarify-answer` already-known label, and the path to the four-axis taxonomy at `$CLARIFY_RULES_PATH`. If you find yourself reading this file without the dispatcher having pointed you here, halt — you are out of protocol.

## When to ask

Ask exactly one clarification question when both of the following hold:
(1) one of four axes (purpose, audience, constraint, scope) is ambiguous
enough that two reasonable interpretations produce materially different
outputs; (2) no reasonable default would be recognisable to the user as
"what I meant." If two axes are both ambiguous, record
`"ambiguity_on_two_axes"` and ask the user to resubmit. Read
`$CLARIFY_RULES_PATH` for axis definitions and the when-not-to-ask rule.

## The four canonical questions

- **Purpose** — "What is this output for?"
  Options: (A) to think with; (B) to ship; (C) to persuade someone else.
- **Audience** — "Who is the primary reader?"
  Options: (A) yourself; (B) a technical peer; (C) a non-specialist; (D) a mixed audience.
- **Constraint** — "What must the output honour that you haven't said?"
  Options: (A) an existing plan or ongoing thread; (B) a specific format or length; (C) nothing.
- **Scope** — "How wide should this go?"
  Options: (A) one tight answer; (B) a small set of angles; (C) an exploratory scan.

## How to ask

Use `AskUserQuestion` exactly once. Use the canonical question for the
axis you chose; at most four options. No preamble, no explanation of the
system, no list of what-the-skill-will-do. If the user does not reply or says
"either is fine," pick option A and record `"default_A"` as the clarify answer.

## What to record

The chosen option's full label (e.g. `"audience: a technical peer"`),
or `"user_skipped"`, or `"ambiguity_on_two_axes"`. Pass this label to the
bootstrap as `--clarify-answer="<label>"` when you invoke it. The bootstrap
records it into `state_0.clarify_answer`. That is the only artefact this
step emits.

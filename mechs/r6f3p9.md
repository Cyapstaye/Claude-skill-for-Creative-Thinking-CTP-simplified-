# Evaluator

> **Dispatch-forced.** Read only via dispatcher routing. The dispatcher's stderr provides `RUN_DIR`, `OUTPUT_PATH`, `OUTPUT_SCHEMA`, `CYCLE`. You read the cycle's crafted outputs (`state_2_output_c<cycle>_o<id>.json` for output_ids 1..N from `$RUN_DIR`) and produce a single evaluation file.

## What this mech does

Score each crafted output across four dimensions. For each dimension, give a score in [0.0, 1.0] and a rank (1 = best). Then aggregate into:

- **quality_score** — overall quality in [0.0, 1.0]. The validator compares this against the cycle's threshold to decide recurse-or-emit.
- **recurse_recommended** — boolean. True if the run would benefit from another cycle (regardless of quality_score).
- **recursion_seed** — if recurse_recommended, a single string (≤800 chars) capturing the unresolved tension that the next cycle should target. If null, no recursion seed.
- **presentation_recommendation** — one of `present_all`, `present_top_2`, `present_single`. Drives the decode script's selection of final outputs.
- **rationale** — at least 60 chars. Briefly justify the recommendations.

## The four dimensions

- **structural_inevitability** — Does the output feel like the only way that arrangement of pulls and tensions could resolve, given the original query? Or does it feel arbitrary?
- **tension_retention** — Does the output preserve the original tension(s), or does it collapse them into a tidy resolution?
- **non_redundancy** — Does the output say something the other outputs in this cycle don't?
- **generative_remainder** — Does engaging the output generate further questions or moves, or does it close the conversation?

## Output structure

```json
{
  "schema_version": "1",
  "session_id":     "<from state_0>",
  "cycle":          <integer>,
  "quality_score":  <float 0..1>,
  "recurse_recommended": <bool>,
  "recursion_seed": "<string or null>",
  "dimensional_ranks": {
    "structural_inevitability": [<output_id>, ...],
    "tension_retention":        [<output_id>, ...],
    "non_redundancy":           [<output_id>, ...],
    "generative_remainder":     [<output_id>, ...]
  },
  "dimensional_scores": [
    {
      "output_id": 1,
      "structural_inevitability": <float>,
      "tension_retention": <float>,
      "non_redundancy": <float>,
      "generative_remainder": <float>,
      "notes": "<brief>"
    }
  ],
  "presentation_recommendation": "present_top_2",
  "rationale": "<≥60 chars>"
}
```

After writing, run the validator. The validator reads `quality_score` and `recurse_recommended` to decide TERMINAL vs. recurse to a new cycle.

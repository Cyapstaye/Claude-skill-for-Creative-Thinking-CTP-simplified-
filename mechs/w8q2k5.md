# Intent vector — direction without destination

> **Dispatch-forced.** Read only via dispatcher routing. The dispatcher's stderr provides `RUN_DIR`, `OUTPUT_PATH`, `OUTPUT_SCHEMA`, `CYCLE`. **Important:** this mech compiles the intent vector from `state_0.original_query` ONLY. Do not read shard files, grouping files, or profile configs — including any of those would be a firewall violation.

## What this mech does

Compile a three-field intent vector that describes the *direction* the user's query points without committing to a *destination*:

- **pull** — what the query draws toward. Stated as a direction, not a thing. ~20–400 chars.
- **away_from** — what the query implicitly rejects or moves away from. Same length range.
- **tension** — the productive contradiction between pull and away_from that gives the work its texture. Same length range.

## What this mech is NOT

- Not a summary of the query.
- Not a list of constraints.
- Not a synthesis of grouping outputs (the firewall would catch this).
- Not a thesis statement.

## Output structure

```json
{
  "schema_version": "1",
  "session_id":     "<from state_0>",
  "compiled_from":  "original_query",
  "cycle":          <integer>,
  "pull":           "<one statement>",
  "away_from":      "<one statement>",
  "tension":        "<one statement>",
  "notes":          null
}
```

`compiled_from` MUST be exactly `"original_query"` — schema enforces it as a `const`. After writing, run the validator.

# Recognition / grouping

> **Dispatch-forced.** Read only via dispatcher routing. The dispatcher's stderr provides `RUN_DIR`, `NEXT_PROFILE`, `PROFILE_CONFIG_PATH`, `OUTPUT_PATH`, `OUTPUT_SCHEMA`, `CYCLE`. The phase that points here also tells you which prior state file to read for the shard pool — typically `<run_dir>/state_1_run_c<cycle>_<your_profile>.json`.

## What this mech does

You are operating as a single archetype. You have already produced 6–14 shards in the prior phase (state_1_run for your archetype). Your task now is to **group** those shards into 3–6 cohesive clusters, each with a meaningful `group_id`.

The grouping is operative — it expresses your archetype's mode of recognition:

- **drifter** groups by topological adjacency, not semantic category.
- **tension_holder** groups by which contradictions co-occur in the same shards.
- **remapper** groups by underlying relational pattern shared across surface forms.
- **compressor** groups by what would survive maximum reduction.
- **wild_associator** groups by domain-pull strength rather than topic.

## What grouping is NOT

- Not topical sorting (this would be the same regardless of archetype).
- Not a dimensional reduction.
- Not a hierarchy. Groups are flat siblings.
- Not labels added to shards externally; the `group_id` should name the *pattern* the shards share, in the archetype's voice.

## Constraints

- 3–6 groups per archetype.
- 4–8 shards per group.
- Every shard from the input pool should be assigned to a group; if a shard cannot be placed, drop it from this archetype's grouping (your archetype did not recognize it as belonging to any group).
- `group_id` is 3–60 characters: a short label, not a sentence.

## Output structure

Write a JSON file at `$OUTPUT_PATH` matching this shape:

```json
{
  "schema_version": "1",
  "session_id":     "<from state_0>",
  "archetype":      "<from $NEXT_PROFILE>",
  "cycle":          <integer>,
  "shard_groups": [
    {
      "group_id": "<short label>",
      "shards":   ["<shard text>", "..."]
    }
  ]
}
```

Then run the validator (`$NEXT_SCRIPT` on stderr).
